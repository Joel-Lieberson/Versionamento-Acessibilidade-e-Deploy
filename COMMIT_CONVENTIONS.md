
# Commits Semânticos (Conventional Commits)

Formato: `tipo(escopo opcional): descrição breve`

Tipos mais usados:
- **feat**: nova funcionalidade
- **fix**: correção
- **docs**: documentação
- **style**: formatação (sem mudança de lógica)
- **refactor**: refatoração
- **test**: testes
- **build**: mudanças de build ou dependências
- **ci**: mudanças de CI
- **chore**: tarefas diversas

Exemplos:
- `feat(spa): adiciona roteador por hash`
- `fix(form): corrige validação de CPF`
- `docs(readme): adiciona passos de deploy`

Use `BREAKING CHANGE:` no corpo do commit quando houver quebra de compatibilidade.
